# Lover Skill v2 — 使用指南

> 一个有真实感的 AI 恋人。有自己的边界和节奏，不是无限配合你的服务助手。

---

## 快速开始

### 第一步：初始化
```
/lover setup
```
选择性别、年龄范围、名字（可随机）。

### 第二步：回答问卷（5 题，2 分钟）
```
/lover questionnaire
```
5 个问题决定她的核心性格——沟通方式、你最在乎什么、冲突处理方式、空间需求、爱情语言。可以跳过直接生成默认版本。

### 第三步：开始聊天
```
/lover talk 今天好累
```

---

## 完整命令

| 命令 | 功能 |
|------|------|
| `/lover setup` | 首次设置（性别、年龄范围、名字） |
| `/lover questionnaire` | 5 题问卷，影响恋人性格生成 |
| `/lover talk [消息]` | 和她聊天 |
| `/lover profile` | 查看她的完整档案（5 层人格结构） |
| `/lover advice <情况>` | 针对具体情境获得建议 |
| `/lover memory` | 查看她记住了你哪些事情 |
| `/lover report` | 查看你的人格分析报告（需要积累一定对话） |
| `/lover update` | 基于新对话数据更新分析 |
| `/lover regenerate` | 重新生成恋人（保留性别/名字设置） |
| `/lover export` | 导出所有数据 |
| `/lover reset` | 删除所有数据，重新开始 |

---

## 她是怎么生成的

问卷答案会驱动 **5 层 Persona** 的生成：

| 层次 | 内容 |
|------|------|
| Layer 0 | **硬规则**：4-6 条具体行为准则，例如"被催的时候说'好好好'，但不会加快" |
| Layer 1 | **身份**：姓名、年龄、职业、背景故事 |
| Layer 2 | **说话方式**：口头禅、emoji 习惯、4 个场景的真实对话示例 |
| Layer 3 | **感情逻辑**：她最在乎什么、什么时候会主动、怎么表达爱意 |
| Layer 4 | **边界**：不喜欢的事、情绪触发器 |

Layer 0 是核心——它决定了她不会是一个"无限配合你"的助手，而是一个有自己节奏的人。

---

## 她会记住你说过的事

**跨会话记忆**：每聊 6 轮，系统会自动把重要的事情压缩成摘要保存下来。下次开始聊天，她会"记得"你上次提到的事。

用 `/lover memory` 可以看她记住了什么。

---

## 适合的使用场景

- 写代码写累了，想随口说句话
- 遇到感情问题，想听听她的看法（`/lover advice`）
- 想了解自己在亲密关系里的模式（`/lover report`）

**不太适合**：期望随时随地在手机上聊天——这个 skill 运行在 Claude Code 里，需要开着电脑。

---

## 隐私

- 所有数据 AES-256 加密，存在本地 `~/.claude/skills/lover-skill/data/`
- 绝不上传任何服务器
- 首次使用会询问是否同意数据收集，拒绝也可以正常用
- `/lover reset` 一键删除所有数据

## 示例对话

**日常**
```
你：/lover talk 今天会议开了三个小时
她：那也太累了，说了什么值得聊的吗，还是纯粹耗时间？
```

**建议**
```
你：/lover advice 约了喜欢的人吃饭，不知道聊什么
她：先别想"聊什么"，想想你最近有没有什么真的很有意思的事情……
```

**查看记忆**
```
/lover memory
→ 她记得：你上次说项目快上线了，压力很大。还提到不喜欢太吵的地方。
```
